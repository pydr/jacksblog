拥抱Docker

科技 July 10, 2018, 2:25 p.m. admin 1000000 Comments
| 虚拟化（Virtualization） 在认识Docker之前，我们首先需要搞清楚一个概念，虚拟化。 什么是虚拟化技术？ 在计算机网络中，虚拟化是一种资源管理技术，是将计算机的各种实体资源，如服务器、网络、内存及存储等予以抽象、转换呈现出来，打破实体间不可分割的障碍，使用户可以以比原本更好的组态方式来应用这些资源。这些资源的的虚拟部分不受有的假设方式、地域或物理组态的限制。 在实际的生产环境中，虚拟化技术主要用来高性能物理硬件产能过剩和老旧硬件产能过低的重组重用，透明化底层物理硬件，从而最大化的利用物理硬件。 虚拟化技术的分类 全虚拟化架构 虚拟机的监视器（hypervisor）是类似于用户的应用程序运行于主机的OS上，如Vmware的workstation，这种虚拟化技术提供了虚拟的硬件。 2. OS层虚拟化架构 OS层虚拟化是依赖于宿主机OS的，与全虚拟化架构不同的是，它不虚拟化硬件，是一种轻量级的虚拟化技术。 3. 硬件层虚拟化 硬件层的虚拟化架构具有高性能和隔离性，因为hypervisor直接在硬件上运行，有利于控制虚拟机访问硬件资源，一般运用于大型虚拟化业务，如目前的云服务器等。 *注解：hypervisor是一种运行在物理服务器和操作系统之间的中间软件层,可允许多个操作系统和应用共享一套基础物硬件，因此也可以看作是虚拟环境中的“元”操作系统，它可以协调访问服务器上的所有物理设备和虚拟机，也叫虚拟机监视器(Virtual Machine Monitor， VMM)。 | Docker 什么是Docker？ Docker诞生于2013年，是一个开源的应用容器引擎，基于 Go 语言 编写，并遵从Apache2.0协议开源。Docker 可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口，更重要的是容器性能开销极低。 Docker的容器技术 在计算机技术中，容器拥有相当漫长且传奇的历史。容器（container）与管理程序虚拟化（hypervisor virtualization，HV）有所不同，管理程序虚拟化通过中间层将一台或多台独立的机器虚拟运行于物理硬件之上，而容器则是直接运行在操作系统内核之上的用户空间。因此，容器虚拟化又称为 操作系统级虚拟化，容器技术可以让多个独立的用户空间运行在t哦那个一台宿主机上，且资源消耗极低，相互之间无影响。 Docker的应用场景 Web 应用的自动化打包和发布。 自动化测试和持续集成、发布。 在服务型环境中部署和调整数据库或其他的后台应用。 从头编译或者扩展现有的OpenShift或Cloud Foundry平台来搭建自己的PaaS环境。 Docker的三个基本组件 1. 镜像（Image） 镜像是Docker的基石，用户基于镜像来运行自己的容器，镜像是Docker声明周期的“构建”部分。通俗来讲，镜像就是用户创建的虚拟化操作系统文件。 2. 容器（Container） 容器是基于镜像创建和启动起来的，容器中可以有一个或多个进程，一旦容器启动后，我们就可以进入容器中操作，就像操作一个实体的操作系统一样。通俗来讲，一个镜像运行起来后就是一个容器。容器是Docker声明周期中的启动或执行阶段。 3. 仓库（Repository） 仓库是用来保存镜像的。就像github中的仓库一样，用户可以上传自己打包的镜像或下载别人创建的镜像来运行。官方的Repositor中提供了丰富的镜像资源。用户注册登录后即可上传下载。 Docker借鉴了标准集装箱的概念。集装箱将货物运送到世界各地，Docker将这个模型运用于自己的设计中，唯一不同的是，集装箱运送的是货物，而Docker运送的是软件，Docker的LOGO也很好的诠释了它的设计理念。 使用Docker，开发者可以快速的构建一个应用程序服务器、一个消息总线、一套使用工具、一个持续继承测试环境或者任意一种应用程序、工具软件。我们可以本地构建一个完整的测试环境，也可以为生产或开发环境快速复制一套应用程序栈。
